# Contributing to Omni-X

Thank you for your interest in contributing to Omni-X!

## Submitting to VoltAgent Awesome Agent Skills

To add this skill to the [VoltAgent Awesome Agent Skills](https://github.com/VoltAgent/awesome-agent-skills) directory:

1. **Fork the Repository**
   - Go to https://github.com/VoltAgent/awesome-agent-skills
   - Click "Fork" to create your own copy

2. **Add Omni-X to the README**
   - Edit the README.md file
   - Add the following entry in the appropriate category (Social Media / Data Extraction):
   ```markdown
   - [Omni-X](https://github.com/HardBrick21/Omni-X) - X (Twitter) data extraction skills for AI agents. Extract profiles, tweets, followers, followings, media, and search results.
   ```

3. **Create a Pull Request**
   - Commit your changes
   - Create a PR with title: "Add Omni-X - X (Twitter) Data Extraction Skills"
   - In the PR description, mention:
     - Clear documentation (SKILL.md, AI_AGENT_GUIDE.md)
     - Standardized interface for AI agents
     - Built-in rate limiting and best practices
     - Active maintenance commitment

4. **Requirements Met**
   - ✅ Clear documentation (SKILL.md as entry point)
   - ✅ Standardized skill interface (TwitterSkillInterface)
   - ✅ Example code (agent_example.py)
   - ✅ Comprehensive guides (AI_AGENT_GUIDE.md, LOGIN_GUIDE.md)
   - ✅ Best practices (rate limiting, error handling)
   - ✅ Disclaimer and terms of use
   - ✅ Active repository with recent commits

## Contributing to Omni-X

### Reporting Issues
- Use GitHub Issues to report bugs or request features
- Provide clear reproduction steps for bugs
- Include relevant error messages and logs

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages (`git commit -m 'Add amazing feature'`)
6. Push to your fork (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Code Style
- Follow PEP 8 for Python code
- Add docstrings to all functions and classes
- Include type hints where appropriate
- Write clear, descriptive commit messages

### Testing
- Test all changes before submitting
- Ensure backward compatibility
- Update documentation if needed

### Documentation
- Update README.md for user-facing changes
- Update SKILL.md for skill interface changes
- Update AI_AGENT_GUIDE.md for API changes
- Keep examples up to date
